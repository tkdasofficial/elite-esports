### PUSH NOTIFICATIONS (Firebase Cloud Messaging)

Set up Firebase Cloud Messaging for push notifications:

1. **Install dependencies**: `expo-notifications`, `@react-native-firebase/app`, `@react-native-firebase/messaging` (or use `expo-notifications` with FCM)
2. **On login/signup**, request notification permissions and get the device FCM token
3. **Save the token** to the `fcm_tokens` table:
   ```js
   await supabase.from('fcm_tokens').upsert({
     user_id: user.id,
     token: fcmToken,
     platform: Platform.OS, // 'android' or 'ios'
     device_info: `${Device.modelName} ${Device.osVersion}`,
     display_name: user.user_metadata?.name || '',
     email: user.email || '',
     updated_at: new Date().toISOString()
   }, { onConflict: 'user_id' });
   ```
4. **Handle incoming notifications**: show in-app alert when app is foreground, navigate to relevant screen on tap
5. **Token refresh**: listen for token refresh events and update fcm_tokens table
6. The backend (Supabase Edge Function) already handles sending push notifications via FCM v1 API — the mobile app only needs to register tokens and handle received notifications

- Create a Firebase project
- Add Android app (download google-services.json → place in android/ folder or configure via app.json)
- Add iOS app (download GoogleService-Info.plist → place in ios/ folder or configure via app.json)

### CRITICAL IMAGE HANDLING RULES

The match and game banners are stored in Supabase Storage. The `banner_url` column stores ONLY the filename/path (e.g., `1234567890.jpg`), NOT the full URL.

To display a banner image, always construct the full URL:
```
https://azxhcalksgudjemwjekd.supabase.co/storage/v1/object/public/game-banners/${banner_url}
```

Create a helper function:
```typescript
const STORAGE_URL = 'https://azxhcalksgudjemwjekd.supabase.co/storage/v1/object/public/game-banners';

function getBannerUrl(path: string | null): string | null {
  if (!path) return null;
  return `${STORAGE_URL}/${path}`;
}
```

Use this everywhere banners are displayed. Always include a fallback placeholder when banner_url is null.
